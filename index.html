为什么叫 Mote's Log

发表于 2026-06-08

从大一开始，我就陆陆续续想过要有一个自己的博客。

后来搭了一个 Hexo，起了名字，写了 hello world。然后就停在那里了。

不是没有东西写。是我还没想清楚自己是谁。

我给自己取了个英文名，叫 Mote。

Mote 的意思是微粒、尘埃，极小，微不足道。

它不是什么自谦。它是我对世界之大的一种确认——正因为知道自己渺小，反而对广阔的空间和无限的可能性有了期待。一粒尘能飘到哪里，这件事本身就值得好奇。

另外还有一层。”尘”这个字，有一部分来自我最喜欢的人的名字。

我的网名是 微尘_Paltry。Paltry 这个词，大概的意思是：我知道这在你的价值排序里微不足道。但那又怎样？我根本不在乎这在你的眼里重不重要。

所以叫 Mote 不是放低自己。是选择了一个位置，然后站在那里。

至于这个博客的名字。

想了很久，有过一堆候选。最后选了 Mote’s Log。

没什么花哨的。Log 就是日志的意思。

不是教程站，不是作品集，不是要教谁什么。

这里是我的 Log。我在这里记录自己。

如果有路人恰好看到，那也很好。但这里首先是写给未来的我的。

大一搭的那个 Hexo 博客还在。两篇旧文章我也不打算删，留着当纪念。

现在是大三下了，距离那次训练营快两年，我变了不少。

所以这里不是”重新开张”。

是从一个新的起点开始写。

之后不定期更新。有感才写。

Mote

基于Hexo的博客编写

发表于 2024-07-11

常用的Hexo命令：

一、初始化和安装

  • hexo init [folder]：在指定文件夹初始化一个新的 Hexo 项目。如果不指定文件夹，默认在当前目录下创建。
  • npm install：在项目文件夹中运行此命令来安装 Hexo 及其依赖。

二、本地开发

  • hexo server（简写 hexo s）：启动本地服务器，预览你的网站。
  • hexo generate（简写 hexo g）：生成静态文件。
  • hexo deploy（简写 hexo d）：部署网站到指定的服务器。
  • hexo clean：清除缓存文件 (db.json) 和已生成的静态文件 (public)。

三、内容管理

  • hexo new [layout] <title>：根据指定的布局（layout），创建一个新的文章。
  • hexo publish [layout] <filename>：发布草稿。

四、其他常用命令

  • hexo list <type>：列出网站中的资料，如：posts, routes 等。
  • hexo version：显示 Hexo 的版本。

五、帮助与插件

  • hexo help：显示帮助信息，列出所有可用的命令和选项。
  • hexo plugin list：列出已安装的插件。
